⚡ Daily Tasks

งานประจำวัน · ทำตอนเช้า ~30 นาที

1. เช็ค ranking ที่เปลี่ยนแปลง
⏱ 10 นาที High On-Page Content

ทำไม: ปัญหา ranking ตกที่เจอเร็วใน 24 ชม. แก้ได้ก่อนลามไปไกล

📋 Step-by-step:

  1. เปิด Dashboard → ตาราง "Top moving keywords"
  2. ดู keywords ที่ตก > 3 อันดับ (สีแดง ↓)
  3. ถ้าเป็น keyword สำคัญ → คลิกดู SERP diff (เว็บใครขึ้นแทน?)
  4. ถ้าสาเหตุชัด (เช่น competitor เขียนใหม่) → สร้าง task ให้ writer refresh content
  5. ถ้าไม่ชัด → ดูใน Audit ว่ามีอะไรเปลี่ยน (CWV ตก? schema หาย?)
💡 ระบบช่วย: Automation "Ranking drop watcher" จะแจ้ง LINE อัตโนมัติเมื่อตก > 5 อันดับ · งานนี้คุณแค่ยืนยัน + assign
2. Review Google Search Console errors
⏱ 5 นาที High Tech

ทำไม: Crawl error/index issue ทำให้ Google มองไม่เห็นหน้า = score Tech ตก

📋 Step-by-step:

  1. เปิด Project Detail → tab "Audit Issues"
  2. ดู section "GSC alerts" — มี alert ใหม่ไหม
  3. หา 404 errors ใหม่ → fix หรือ redirect
  4. หา "Crawled but not indexed" → เช็คว่า content ดีพอ ไม่ใช่ thin
  5. กด "Submit for indexing" สำหรับหน้าใหม่ที่ยังไม่ index
เป้าหมาย: 0 critical errors · indexed/discovered ratio > 95%
3. เช็ค comments/approvals จากลูกค้า
⏱ 10 นาที Medium Content

ทำไม: ลูกค้า approve ช้า = content publish ช้า = score Content ขึ้นช้า

📋 Step-by-step:

  1. เปิด Content Pipeline → คอลัมน์ "In Review"
  2. เช็คว่ามี comment ใหม่จากลูกค้าหรือไม่
  3. ถ้าลูกค้าขอแก้ → แจ้ง writer ทันที (ไม่ทิ้งข้ามวัน)
  4. ถ้า approve แล้ว → ระบบ publish อัตโนมัติ ตรวจดูว่า publish สำเร็จ + schema ใส่แล้ว
  5. ถ้าลูกค้าเงียบ > 3 วัน → ส่ง reminder ทาง LINE
4. Monitor automation alerts
⏱ 5 นาที Medium Tech

ทำไม: Automation จับปัญหาให้แล้ว แต่บางอย่างต้องมีคนตัดสินใจต่อ

📋 Step-by-step:

  1. เปิด Automations → ดู "Run history" 24 ชม.ที่แล้ว
  2. เช็ค rule ที่ status = "failed" หรือ "needs-action"
  3. ถ้า fail บ่อยๆ → debug rule (โอเค ปรับ threshold หรือ disable)
  4. ถ้า task ถูกสร้างให้ specialist → ตรวจ priority ถูกหรือยัง
5. เช็ค AI Visibility ลูกค้าเด่นๆ
⏱ 5 นาที High AI Vis

ทำไม: AI citations เป็น ranking factor ใหม่ที่คู่แข่งยังไม่ทำ · ปรับเล็กน้อยได้ผลเยอะ

📋 Step-by-step:

  1. เปิด AI Visibility → ดู prompt portfolio ของลูกค้าที่ดูแล
  2. เช็ค prompts ที่ เคย cited → กลายเป็น mention/— ใน 7 วัน
  3. ถ้าหลุดเยอะ → คลิก "Why not cited?" diff explainer
  4. สร้าง task ให้ writer refresh content หรือเพิ่ม schema
💡 Automation: "AI citation lost" rule จะ auto-create brief ให้แล้ว · งานคุณคือ verify + assign

📅 Weekly Tasks

งานสัปดาห์ละครั้ง · ทำวันจันทร์ ~3 ชม.

1. รัน full site audit ของลูกค้าทุกราย
⏱ 30 นาทีHighOn-PageTech

ทำไม: Snapshot สถานะเว็บ — ดูว่ามีปัญหาใหม่เกิดขึ้นในสัปดาห์ที่ผ่านมา

📋 Step-by-step:

  1. กด "Run audit" ในทุก project (Automation จะรันอยู่แล้ว แต่ manual run ให้ได้ snapshot ปัจจุบัน)
  2. ดู "Audit Issues" → เปรียบเทียบกับสัปดาห์ที่แล้ว
  3. Critical issues ใหม่ → สร้าง task priority high
  4. ถ้า score ตก > 5 จุด → ดูว่ามิติไหนเสีย แล้วโฟกัสที่นั่น
2. Backlink analysis — ใหม่ + หาย
⏱ 20 นาทีHighOff-Page

ทำไม: Backlinks ที่หายไป (lost links) ทำให้ Off-Page score ตก · backlinks ใหม่ DA สูงเพิ่ม authority

📋 Step-by-step:

  1. เปิด Project Detail → tab "Backlinks"
  2. ดู "Lost links" 7 วัน → ถ้าเป็น link สำคัญ → ติดต่อขอกลับ หรือทดแทนด้วย link ใหม่
  3. ดู "New links" → ถ้าเป็น toxic (spam/porn) → disavow
  4. เก็บ list backlink ใหม่ DA > 50 เพื่อ pitch case study ต่อ
หมายเหตุ: ระบบเรา sync DataForSEO ทุก 24 ชม. · ข้อมูลเป็น near-realtime
3. Local Grid scan สำหรับ local clients
⏱ 15 นาทีHighLocal

ทำไม: Local ranking ต่างกันมากในแต่ละพื้นที่ · Grid scan เห็นภาพชัด

📋 Step-by-step:

  1. เปิด Local Grid สำหรับลูกค้า local (ร้านอาหาร, spa, คลินิก)
  2. รัน 7×7 grid สำหรับ keyword หลัก
  3. เปรียบเทียบกับสัปดาห์ที่แล้ว — top 3 coverage เพิ่ม/ลด?
  4. หา weak zones (สีแดง/ส้ม) → focus area สำหรับ GBP optimization
  5. แจ้งลูกค้าให้ขอ review จากคนในพื้นที่ weak zone
4. Content brief queue — สร้าง brief ใหม่
⏱ 40 นาทีHighContent

ทำไม: Writer ต้องมี brief จ่ายอย่างต่อเนื่อง · queue ว่าง = ไม่มี content publish = score ตก

📋 Step-by-step:

  1. ดู content calendar — ลูกค้าไหนใกล้หมด brief
  2. เปิด Keyword Planner → ใส่ topic ของลูกค้านั้น
  3. เลือก keyword ที่ KD ต่ำ-ปานกลาง + volume ดี (sweet spot: KD 20-50, vol > 500)
  4. กด "สร้าง Brief จาก keyword" → ระบบ generate outline ให้
  5. review brief + เพิ่มเงื่อนไขพิเศษ → assign writer
เป้าหมาย: มี brief ใน pipeline ≥ 4 ชิ้นต่อลูกค้า ตลอดเวลา
5. Internal linking audit + เพิ่ม links
⏱ 30 นาทีHighOn-Page

ทำไม: Internal linking เร็วและฟรี — ส่ง link equity ระหว่างหน้า เพิ่ม ranking ของ pillar

📋 Step-by-step:

  1. ใช้ Internal Linking suggester ของระบบ
  2. หา orphan pages (ไม่มี link ชี้มา) → เพิ่ม links จากหน้าที่เกี่ยวข้อง
  3. หา pillar pages (หน้าใหญ่) → เพิ่ม link จาก sub-articles 3-5 ตัว
  4. ตรวจ anchor text — ห้ามใช้ "click here" ให้ใช้ keyword ที่เกี่ยวข้อง
  5. publish updates ตรง (ไม่ต้อง wait approval สำหรับ link เล็กๆ)
📈 Impact: ทำต่อเนื่อง 4 สัปดาห์ = score On-Page เพิ่ม 3-8 จุด
6. Schema validation check
⏱ 15 นาทีMediumTechAI Vis

ทำไม: Schema ที่ error = Google ไม่อ่าน + AI ไม่ใช้ในการ quote

📋 Step-by-step:

  1. ดู Audit Issues → section "Structured Data"
  2. เช็คทุก schema ที่มี error/warning
  3. ใช้ Google Rich Results Test เพื่อ verify
  4. ถ้าหน้าไหนยังไม่มี schema ที่ควรมี → ใช้ Schema Generator สร้าง
  5. ตรวจว่า dev/CMS ใส่ schema ถูกที่
7. CWV diagnostic + dev tickets
⏱ 20 นาทีMediumCWV

ทำไม: CWV ตกค่อยๆ จาก third-party scripts/รูปใหม่ที่ใส่ — ต้องคอยจับ

📋 Step-by-step:

  1. เปิด Project → tab CWV
  2. เปรียบเทียบ INP/LCP/CLS กับสัปดาห์ที่แล้ว
  3. หาหน้าที่ INP > 200ms — debug script ที่หนัก
  4. สร้าง dev ticket ใน ClickUp/Asana (auto-sync จากระบบ)
  5. แจ้ง dev team พร้อม recommendation
8. Update content calendar
⏱ 20 นาทีMediumContent

ทำไม: ลูกค้าต้องการความสม่ำเสมอ · Google มอง freshness

📋 Step-by-step:

  1. เปิด Content Pipeline · ดู kanban
  2. วางแผน 4 สัปดาห์ข้างหน้า — 1 pillar + 2-3 sub-articles ต่อลูกค้า
  3. match กับเทศกาล/event (สงกรานต์, ปีใหม่, Black Friday)
  4. set due dates + assign writers
9. Refresh old content (1-2 articles)
⏱ 45 นาทีHighContent

ทำไม: เนื้อหา 1-2 ปีเก่า rank ตกค่อยๆ · refresh เร็วและถูกกว่าเขียนใหม่ + Google ชอบ freshness

📋 Step-by-step:

  1. หาบทความที่: published > 12 เดือน + เคย rank ดี + ตอนนี้ตก
  2. อ่านบทความปัจจุบันที่ติด top 3 — เขาเพิ่มอะไรที่เราขาด?
  3. เพิ่ม section ใหม่, อัปเดต stats/ปี, refresh examples
  4. เพิ่ม author byline ถ้ายังไม่มี (E-E-A-T)
  5. publish + update dateModified ใน schema
  6. submit re-index ใน GSC
💡 Pro tip: Refresh 1 article สร้าง traffic ได้เท่าๆ กับเขียนใหม่ 3 articles · ทำสัปดาห์ละครั้งสม่ำเสมอ
10. Competitor SERP analysis
⏱ 25 นาทีMediumContent

ทำไม: เข้าใจคู่แข่ง = หา content gap = topics ที่ทำง่ายและ rank ได้

📋 Step-by-step:

  1. เลือก competitor 2-3 ราย
  2. ดู keyword ที่เขา rank ดี (top 10) แต่เราไม่
  3. filter ด้วย KD ≤ 50 + volume > 300
  4. เพิ่มเป็น content brief ใน queue
11. ส่ง weekly digest ให้ลูกค้า
⏱ 15 นาทีLow

ทำไม: รักษาความสัมพันธ์ + ลูกค้าเห็น value ของ retainer

📋 Step-by-step:

  1. ระบบ generate weekly digest อัตโนมัติทุกจันทร์ 9:00
  2. คุณ review + เพิ่ม commentary ส่วนตัว
  3. highlight 1 win + 1 issue ที่กำลังแก้
  4. ส่ง LINE OA + email

📆 Monthly Tasks

งานเดือนละครั้ง · ใช้เวลา ~1 วันรวม · ทำต้นเดือน

1. Monthly report — ทุกลูกค้า
⏱ 1 ชม. (auto + review)Low

ระบบ generate PDF อัตโนมัติทุกวันที่ 1 · AM review + เพิ่ม commentary · ส่งอีเมล + LINE OA

📋 Step-by-step:

  1. เปิดทุก draft report ที่ระบบ generate
  2. review AI Summary — ปรับโทนให้เหมาะกับลูกค้า
  3. เพิ่ม narrative ของ AM ส่วนตัว (1-2 ย่อหน้า)
  4. กด "Send" → ระบบส่ง email + LINE + log
2. Full technical SEO deep dive
⏱ 2 ชม.HighTech

เช็คทุก technical aspect ที่ไม่ได้ดูทุกสัปดาห์

  1. sitemap.xml — มีหน้าครบ + submit GSC?
  2. robots.txt — block สิ่งที่ไม่ควร? อนุญาต GPTBot/ClaudeBot?
  3. hreflang — ถ้ามีหลายภาษา ใส่ครบ?
  4. HTTPS — ทุกหน้า + redirect HTTP → HTTPS
  5. Canonical chains — ไม่มี loop หรือ chain > 2
  6. JS rendering parity — Google เห็นตรงกับ browser?
  7. Mobile responsiveness — ทุก breakpoint
  8. 404/410/redirect — ไม่มี broken
3. Keyword research refresh
⏱ 1.5 ชม.HighContent
  1. ดู keyword list ปัจจุบัน — มีตัวที่เลิก track ได้ไหม (volume ตก)
  2. หา new keyword opportunities จาก GSC: "queries with impressions but no clicks"
  3. ทำ keyword expansion ใหม่จาก seed keyword ของลูกค้า
  4. filter: KD ≤ 50, volume ≥ 200, intent matches business
  5. เพิ่มเข้า tracker
4. AI Visibility deep dive
⏱ 1 ชม.HighAI Vis
  1. refresh prompt portfolio — เพิ่ม 10-20 prompts ใหม่ที่เกี่ยวข้องกับ business
  2. วิเคราะห์ SOV trend 30 วัน — เพิ่ม/ลด?
  3. identify content gaps จาก "why not cited" diff
  4. วาง content strategy เพื่อชนะ citations ในไตรมาสถัดไป
5. Backlink outreach campaign
⏱ 3 ชม.HighOff-Page
  1. identify backlink prospects — เว็บ industry ที่เขียนเรื่องคล้ายๆ
  2. research contact + email
  3. เตรียม pitch (guest post, link building, broken link replacement)
  4. ส่ง 10-20 emails
  5. track responses (เป้า: 1-2 backlinks/เดือนจาก campaign นี้)
6. Schema upgrade pass
⏱ 45 นาทีMediumTechAI Vis
  1. identify หน้าใหม่ที่ขาด schema
  2. upgrade schema เก่าให้ chained (Product → Org → Person sameAs)
  3. เพิ่ม Author entity graph สำหรับ content E-E-A-T
  4. test ทุก schema ใหม่กับ Rich Results Test
7. Re-optimize underperforming pages (3-5 หน้า)
⏱ 2 ชม.HighOn-PageContent

หน้าที่ติด page 2 (11-20) — ใกล้ top 10 มาก, ปรับนิดเดียวขยับขึ้นได้

  1. filter GSC: pages with avg position 11-20 + impressions ≥ 500
  2. ดู NLP score — ขาด entity/term ไหน
  3. เพิ่ม content depth (FAQ section, comparison table, expert quote)
  4. เพิ่ม internal links จากหน้าอื่น 3-5 ลิงก์
  5. อัปเดต title/meta ใหม่ให้ดึงดูดคลิก
  6. publish + re-submit GSC
📈 Impact ตัวจริง: 30-50% ของหน้าที่ทำแบบนี้ขึ้น top 10 ใน 30 วัน
8. GBP posts + Q&A refresh (local clients)
⏱ 30 นาทีMediumLocal
  1. โพสต์ใน GBP 4 ครั้ง/เดือน (offer, event, what's new, product)
  2. ตอบ Q&A ใหม่ทั้งหมด
  3. request reviews จากลูกค้าใหม่
  4. upload รูปใหม่ 2-3 รูป
9. Review automation rules
⏱ 30 นาทีLow
  1. ดู Automation success rate (เป้า ≥ 99%)
  2. rule ที่ trigger เยอะแต่ไม่มี action จริง → ปรับ threshold
  3. เพิ่ม rule ใหม่จาก pattern ที่เกิดบ่อยๆ (ทำซ้ำเอง 3 ครั้งขึ้นไป)
10. Content performance review
⏱ 1 ชม.MediumContent
  1. ดูบทความที่ publish ในเดือนที่แล้ว
  2. top performers — ทำไมได้ traffic เยอะ? เลียนแบบสำหรับเดือนต่อ
  3. under-performers — schedule refresh ใน 6 เดือน
  4. NLP score ต่ำ → list ไว้สำหรับ improve
11. Update sitemap.xml + submit GSC
⏱ 15 นาทีLowTech
ตรวจให้แน่ใจ sitemap มีทุกหน้าใหม่ที่ publish ในเดือนที่ผ่านมา + re-submit ใน GSC
12. Strategic review meeting กับลูกค้า
⏱ 1 ชม./ลูกค้าMedium
นัด Zoom/Google Meet กับลูกค้า · ทบทวน monthly report · วาง strategy เดือนถัดไป
13. AI Crawler access check
⏱ 20 นาทีMediumAI Vis
เช็คว่า WAF/Cloudflare ไม่ block GPTBot, ClaudeBot, PerplexityBot · ใช้ AI Crawler Access Auditor ในระบบ
14. Team retrospective
⏱ 1 ชม.Low
ประชุมทีมเดือนละครั้ง · review wins/issues · improve process · share findings

เพิ่ม schema markup สำหรับ Article (top 10 หน้า)

หน้าที่มี traffic เยอะ ยังไม่มี schema · ใช้ Schema Generator · ผลภายใน 1-2 สัปดาห์ AI ดึงไปใช้

⏱ 45 นาที+5-10 AI VisTechAI Vis

เพิ่ม Author entity ทุก content

สร้าง /authors/ pages พร้อม sameAs (LinkedIn, Twitter) · ใส่ Person schema · ส่งผลต่อ E-E-A-T ทันที

⏱ 1 ชม.+5-8 ContentContentAI Vis
4

Fix broken internal links

ใช้ Broken Link Sweeper · automation จะ suggest redirect ให้ · กดยืนยัน publish

⏱ 20 นาที+2-4 scoreTech
5

เพิ่ม FAQ section ที่ทุกหน้าหลัก

5-7 คำถามที่ลูกค้าจริงๆ ถาม · ใส่ FAQPage schema · ติด People Also Ask + AI Overview

Compress รูปภาพหนัก (LCP fix)

⏱ 30 นาที+3-6 CWVCWV

เติม alt text ที่ขาด

หา title ที่ > 60 (ถูกตัด) หรือ < 30 (สั้นไป) · rewrite + ใส่ keyword หลัก
